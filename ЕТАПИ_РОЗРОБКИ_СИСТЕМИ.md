# Етапи розробки системи: BoardGamesLibrary

Документ описує розробку цього проєкту від початкової ідеї до завершеного продукту.  
Структура витримана за шаблоном: **Етап -> Опис -> Код -> Пояснення**.

---

## Технологічний стек

**Основа:**

- C# / .NET 8
- WPF (Desktop UI)
- xUnit (тестування)

**Проєктні модулі:**

- `BoardGamesLibrary.Core` — контракти і базові моделі
- `BoardGamesLibrary.Games` — логіка ігор (Chess, Checkers, Reversi, Connect Four)
- `BoardGamesLibrary.Logging` — логування та обробка помилок
- `BoardGamesLibrary.UI` — консольний інтерфейс
- `BoardGamesLibrary.Desktop` — WPF інтерфейс
- `BoardGamesLibrary.Tests` — юніт-тести

**Архітектура:**

- Модульна, з чітким поділом відповідальностей
- Інтерфейсно-орієнтований дизайн (`IGame`, `IBoard`, `IMove`, `IPiece`, `IAIPlayer`)
- Єдина модель стану гри та клонування для AI-симуляцій

## Етап 1. Формування структури рішення

На старті проєкт розбитий на незалежні модулі: ядро, логування, ігрова логіка, 2 UI-шари та тести.

### Базова структура

```text
BoardGamesLibrary/
├── BoardGamesLibrary.Core/
├── BoardGamesLibrary.Logging/
├── BoardGamesLibrary.Games/
│   ├── Chess/
│   ├── Checkers/
│   ├── Reversi/
│   └── ConnectFour/
├── BoardGamesLibrary.UI/
├── BoardGamesLibrary.Desktop/
└── BoardGamesLibrary.Tests/
```

**Пояснення:** така структура дозволяє розвивати логіку ігор незалежно від конкретного UI і повторно використовувати ядро між консоллю та WPF.

## Етап 2. Проєктування ядра (Core)

Визначено єдині контракти для всіх ігор: дошка, стан гри, хід, переможець, валідація, клонування.

### Контракт гри

```csharp
public interface IGame
{
    IBoard Board { get; }
    GameState State { get; }
    Player CurrentPlayer { get; }
    bool MakeMove(IMove move);

    bool IsValidMove(IMove move);

    bool IsGameOver();

    IGame Clone();

```

### Базова дошка

```csharp
public abstract class BaseBoard : IBoard
{
    protected readonly IPiece?[,] _pieces;
    public int Rows { get; }
    public int Columns { get; }

    public virtual IPiece? GetPiece(Position position) { ... }
    public virtual void SetPiece(Position position, IPiece? piece) { ... }
    public virtual bool IsValidPosition(Position position) => position.IsWithinBounds(Rows, Columns);
    public virtual void Clear() { ... }

```

**Пояснення:** контракти забезпечили універсальність: будь-яка гра реалізує однаковий API, а UI працює зі спільною моделлю без дублювання.

## Етап 3. Реалізація системи логування і стабільності

Додано багаторівневе логування (`ConsoleLogger`, `FileLogger`, `CompositeLogger`) та безпечну роботу з файлами логів.

### Надійне відкриття лог-файлів

```csharp
private static StreamWriter? TryCreateWriter(string path)
{
    const int maxAttempts = 5;
    for (int attempt = 0; attempt < maxAttempts; attempt++)
    {
        try
        {
            var stream = new FileStream(
                path,
                FileMode.Append,
                FileAccess.Write,
                FileShare.ReadWrite | FileShare.Delete);
            return new StreamWriter(stream) { AutoFlush = true };
        }
        catch (IOException) when (attempt < maxAttempts - 1)
        {
            Thread.Sleep(50);
        }
    }
    return null;
}
```

**Пояснення:** реалізовано retry + file sharing + fallback-файл. Це прибрало краші при блокуванні логів і дозволило запускати гру стабільно навіть при паралельних екземплярах.

## Етап 4. Реалізація ігрової логіки (Chess, Checkers, Reversi, Connect Four)

На цьому етапі закладено повні правила кожної гри, валідацію ходів, завершення партії, визначення переможця.

### Приклад виконання ходу в шахах

```csharp
public bool MakeMove(IMove move)
{
    if (move is not ChessMove chessMove) return false;
    if (State != GameState.InProgress) return false;
    if (chessMove.Player != CurrentPlayer) return false;
    if (!IsValidMove(chessMove)) return false;

    return _errorHandler.ExecuteSafely(() =>
    {
        var board = (ChessBoard)Board;
        var piece = board.GetPiece(chessMove.From);
        if (piece == null) return false;

        // ... рокіровка, en passant, promotion ...

        _moveHistory.Add(chessMove);
        CheckGameState();
        if (State == GameState.InProgress)
            CurrentPlayer = CurrentPlayer.GetOpponent();
        return true;
    }, false, "Виконання ходу в шахах");
}
```

**Пояснення:** кожен хід проходить ланцюг перевірок (стан гри, власник фігури, валідність, безпечність), після чого оновлюється стан і тільки тоді змінюється гравець.

## Етап 5. Додавання AI з рівнями складності

Реалізовано гру проти комп'ютера з 3 рівнями: Easy, Medium, Hard.

### Вибір алгоритму за складністю

```csharp
public override IMove? ChooseMove(IGame game)
{
    if (game is not ChessGame chessGame) return null;

    return Difficulty switch
    {
        AIDifficulty.Easy => GetRandomMove(chessGame),
        AIDifficulty.Medium => GetBestMoveByHeuristic(chessGame, EvaluatePosition),
        AIDifficulty.Hard => GetBestMoveMinimax(chessGame, 3),
        _ => GetRandomMove(chessGame)
    };
}
```

**Пояснення:** модель складності будується на поступовому ускладненні: випадкові ходи -> евристика -> minimax + alpha-beta pruning.

## Етап 6. Консольний інтерфейс

Побудовано повний консольний цикл: меню, запуск гри, введення ходів, показ результату, повернення в меню.

### Приклад основного меню

```csharp
while (true)
{
    Console.WriteLine("1. Шахи");
    Console.WriteLine("2. Шашки");
    Console.WriteLine("3. Reversi/Othello");
    Console.WriteLine("4. Connect Four");
    Console.WriteLine("0. Вихід");

    {
        case "1": PlayChess(); break;
        case "2": PlayCheckers(); break;
        case "3": PlayReversi(); break;
        case "4": PlayConnectFour(); break;
        case "0": return;
    }
}
```

**Пояснення:** консольна версія стала базовим функціональним шаром для швидкої перевірки логіки без графічного UI.

## Етап 10. Endgame UX: явна індикація перемоги/поразки

На завершення партії додано чіткий флоу: результат + вибір наступної дії.

### Діалог після завершення гри

```csharp
private void HandleGameOverDialog()
{
    if (_game == null || _endgameDialogShown) return;
    _endgameDialogShown = true;

    var winner = _game.GetWinner();
    var resultText = winner switch
    {
        null => "Нічия!",
        _ when _playVsAI && winner.Value == Player.Player1 => "Перемога!",
        _ when _playVsAI && winner.Value == Player.Player2 => "Поразка!",
        _ => $"Переміг: {GetPlayerDisplayName(winner.Value)}"
    };

    var choice = MessageBox.Show(
        this,
        $"{resultText}\n\nЗіграти ще раз?\n\nТак — нова гра\nНі — повернутися в меню",
        "Гру завершено",
        MessageBoxButton.YesNo,
        MessageBoxImage.Question);

    if (choice == MessageBoxResult.Yes) StartNewGame();
    else Close();
}
```

**Пояснення:** додано одноразовий показ (`_endgameDialogShown`) і контроль ефектів (`_endgameEffectsPlayed`), щоб уникнути повторних попапів і дублювання звуків.

## Етап 11. Тестування логіки

Покрито критичні сценарії: старт гри, валідні/невалідні ходи, безпека короля, клонування стану, рокіровка.

### Приклад юніт-тесту

```csharp
[Fact]
public void ValidPawnMove_IsAccepted()
{
    var game = new ChessGame(CreateLogger());
    game.StartNewGame();

    var move = new ChessMove(new Position(1, 0), new Position(3, 0), Player.Player1);
    Assert.True(game.IsValidMove(move));
    Assert.True(game.MakeMove(move));
    Assert.Equal(Player.Player2, game.CurrentPlayer);
}
```

**Пояснення:** тести зафіксували інваріанти і запобігають регресіям при рефакторингу правил.

## Етап 12. Збірка, запуск і реліз

Останній етап — стабільний запуск на Windows без ручного встановлення runtime на цільовій машині.

### Self-contained Desktop збірка

```xml
<PropertyGroup>
  <TargetFramework>net8.0-windows</TargetFramework>
  <UseWPF>true</UseWPF>
  <SelfContained>true</SelfContained>
  <RuntimeIdentifier>win-x64</RuntimeIdentifier>
  <PublishSingleFile>true</PublishSingleFile>
  <IncludeNativeLibrariesForSelfExtract>true</IncludeNativeLibrariesForSelfExtract>
</PropertyGroup>
```

### Батч-запуск із перевірками

```bat
dotnet build BoardGamesLibrary.Desktop\BoardGamesLibrary.Desktop.csproj -c Release
if errorlevel 1 exit /b 1
if exist "BoardGamesLibrary.Desktop\bin\Release\net8.0-windows\win-x64\BoardGamesLibrary.Desktop.exe" set "EXE=..."
start "" "%EXE%"
```

**Пояснення:** автоматизовано локальний запуск, зменшено кількість ручних дій і помилок середовища.
